Глава 2. Выборка jQuery

Базовые правила

В JS есть собственный API для работы с элементами DOM с таким функциями как getElementById() и getElementsByClassName(), предназначенных для работы с определенным типом селектора. Однако эти функции поддерживаются не всеми браузерами. В этом случае jQuery приходит на помощь, т.к. эта библиотека использует использует нативный метод, если он поддерживается браузером и свой, если нет. Поэтому с jQuery не приходится надеяться на поддержку со стороны браузеров.

Более того, между селекторами jQuery и селекторами CSS3 нет никаких несовместимостей.

Элементы в коллекции jQuery будут храниться в том порядке, в котором они появляются в дереве DOM.

Пример Описание Аналог в JS Есть в CSS
* Все элементы на странице getElementsByTagName("*") Да
#special-id Элемент со значением ID special-id getElementsById("special-id") Да
.special-class Элемент с классом special-class getElementsByClassName('special-class') Да
a Все элементы ссылок (a) getElementsByTagName("a") Да
a.special-class Все элементы ссылок (a), у которых есть класс special-class Да
.class.special-class Все элементы с классом class и классом special-class Да
type1, class2 Все элементы селектированные type1 и все элементы селектированные и class2 Да

    Пример: поиск всех сестринских элементов

    Мы можем искать элементы, указав элементом уровень поиска, т.е., другими словами, искать сестринские элементы:

    <script>
    // для начала облегчим себе жизнь, написав функцию, которая
    //принимает в качестве аргумента id списка на странице, а

    //jquery-запроса
    function ol_writer(ol_id, query, n=5){
        ol_id=jQuery(ol_id)
        query=jQuery(query)

            ol_id.append('<li>'+query.eq(i).html().replace('<','<')+"</li>")
        }
    }
    ol_writer('#ex3', '#ex3 ~ *', 3)
</script>

            Фильтры-потомки

            Иногда возникает необходимость выбрать определенного потомка элемента. Часто, однако фильтры потомков путаются с потомками порядка. В частости :eq() и :nth-child(). Если используется первый, то подсчитываются только элементы, соответствующие прикрепленному к псевдоклассу селектору, независимо от числа сестринских элементов перед ним. При использовании же второго подсчитываются все потомки, содержащего элемента независимо от их типа. Другое важное отличие: :nth-child() выводится из спецификации CSS и потому индексирование начинается с 1.

            Селектор Описание Есть в CSS
            :first-child Первый элемент-потомок Да
            :last-child Последний элемент потомок Да
            :first-of-type Первый элемент-потомок заданного типа Да
            :last-of-type Последний элемент-потомок заданного типа Да
            :nth-child(n), :nth-child(even|odd), :nth-child(Xn+Y) n-ый элемент-потомок указанного порядка Да
            :nth-last-child(n), :nth-last-child(even|odd), :nth-last-child(Xn+Y) n-ый элемент-потомок указанного порядка считая от последнего Нет
            :nth-of-type(n), :nth-of-type(even|odd), :nth-of-type(Xn+Y) n-ый элемент-потомок указанного порядка указанного типа Да
            :nth-last-of-type(n), :nth-last-oftype(even|odd), :nth-last-of-type(Xn+Y) n-ый элемент-потомок указанного порядка указанного типа считая от последнего Да
            :only-child Все элементы, у которых нет сестринских элементов Да
            :only-of-type Все элементы, у которых нет сестринских элементов того же типа Да

            Пользовательские фильтры

            Вы можете осуществлять один и тот же выбор, а затем одну и ту же фильтрацию извлеченного набора много раз, задействуя конструкции циклов и выборов. В подобных ситуациях можно создать ярлык для сбора узлов DOM или, выражаясь точнее, создать пользовательский фильтр.

            Чтобы объяснить способ создания пользовательского фильтра, начнем с примера. Представьте: вы разрабатываете техническую игру. У вас есть список уровней, которые следует пройти с определенной степенью сложности, количество очков, которые может заработать пользователь, а также перечень технологий для ее успешного прохождения. Ваш гипотетический список может выглядеть примерно так:

            <ul class="levels">
    <li data-level="1" data-points="1" data-technologies="javascript node
 grunt">Уровень 1</li>
    <li data-level="2" data-points="10"
        data-technologies="php composer">Уровень 2</li>
    <li data-level="3" data-points="100" data-technologies="jquery
 requirejs">Уровень 3</li>
    <li data-level="4" data-points="1000" data-technologies="javascript jquery
 backbone">Уровень 4</li>
</ul>

            Теперь представьте, что нужно часто выбирать уровни (data-level) выше, чем 2, но только тогда, когда они позволяют вам заработать больше чем 100 очков (data-points) и имеют jQuery в списке технологий для использования (data-technologies). Вы уже знаете, как искать элементы li, имеющие ключевое слово jquery внутри атрибута data-technologies (li[data-technologies~="jquery"]).

            Но как вы будете сравнивать числа с помощью селекторов? Правда в том, что вы не можете этого сделать. Чтобы достичь цели, необходимо перебрать изначальный выбор, а затем сохранить только нужные вам элементы, как показано здесь:

            <script> var $levels = $('.levels li[data-technologies~="jquery"]'); //первоначальный ыбор с использоанием селектора атрибута var matchedLevels = []; for (var i = 0; i < $levels.length; i++) { //проход по совпадающему набору элементов if ($levels[i].getAttribute('data-level') > 2 && $levels[i].getAttribute('data-points') > 100) { //проверка соответстует ли текущий элемент требоаниям matchedLevels.push($levels[i]); //добаление к финальному набору элементов } } </script>

            Вместо многократного повторения можно создать пользоательский фильтр:

            <script>
    //фильтр объявляется с помощью функции createPseudo
    $.expr[':'].requiredLevel = $.expr.createPseudo(function (filterParam) {
        return function (element, context, isXml) {
        //возврат анонимной функции, вызванной для выполнения тестов
            return element.getAttribute('data-level') > 2 && element.getAttribute('data-points') > 100;
            //тестирует текущий элемент
        };
    });
</script>

            В функцию createPseudo() мы передаем анонимную функцию, где объявляем параметр filterParams. Внутри этой анонимной функции мы создаем другую и она будет возвращена. Внутренней функции jQuery передает элементы, которые будут обрабатываться по одному за раз (параметр element), DOMElement или DOMDocument. Из них будет производиться выбор (параметр context), и булево значение, определяющее, работаете ли вы в XML-документе или нет (параметр isXML). В середине самой внутренней функции вы пишете код, чтобы проверить, должен ли сохраняться элемент.

            Представьте, что вы хотите выбрать уровни на основе предложенного количества баллов — например, «выбрать все уровни с количеством очков выше чем X». Этот большой X является хорошей возможностью использовать параметр для передачи вашему псевдоселектору. Основываясь на данном требовании, можно создать новый фильтр:

            <script>
    $.expr[':'].pointsHigherThan = $.expr.createPseudo(function (filterParam) {
        var points = parseInt(filterParam, 10);
        //кэширование аргумента для доступности в закрытой внутренней функции
        return function (element, context, isXml) {
            return element.getAttribute('data-points') > points;
            //использование кэшированного аргумента в тесте
        };
    });
</script>

            В итоге, если мы хотим получить все уровни, которые позволяют вам заработать более 50 очков, то можете написать получение двух последних элементов списка:

            <script>var $elements = $('.levels li:pointsHigherThan(50)');</script>

            jQuery существенно упрощает работу с группами переключателей. Чтобы увидеть выигрыш от использования jQuery, достаточно сравнить 2 фрагмента кода, который вытягивает значение набора переключателей:

            //значение набора переключателей на чистом JS
var checkedValue;
var elements = document.getElementsByTagName('input');
for (var i = 0; i < elements.length; i++) {
    if (elements[i].type === 'radio' && elements[i].name === 'some-radio-group' && elements[i].checked) {
        checkedValue = elements[i].value;
        break;
        }
    }
            //значение набора переключателей на jQuery
var checkedValue=jQuery('input:radio[name="some-radio-group"]:checked').val();
